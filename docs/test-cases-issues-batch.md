## 1. #114 空音频输出热词

### 测试前提
- 使用本地版 Type4Me（Qwen3-ASR）
- 热词列表中有内容（设置 > 热词）

### 用例 1.1: 空音频不输出热词
1. 按住热键开始录音
2. **不说话**，立即松开热键（<0.5秒）
3. **预期**: 无任何文本输出，不应出现热词列表
4. **验证**: 输入框为空，历史记录不新增条目

### 用例 1.2: 极短噪音不输出热词
1. 按住热键开始录音
2. 对着麦克风轻敲桌面，立即松开
3. **预期**: 无文本输出或空字符串，不应出现热词
4. **验证**: 同上

### 用例 1.3: 正常语音仍能识别热词
1. 按住热键说 "我在用 Claude 写 TypeScript"
2. 松开热键
3. **预期**: 正常转录，且 Claude、TypeScript 等热词被正确识别
4. **验证**: 识别结果中包含正确的热词拼写

## 2. #111 剪贴板竞态

### 测试前提
- 开启"保留剪贴板"设置（默认开启）
- 准备一个文本编辑器（如 VS Code、备忘录）

### 用例 2.1: 正常注入后剪贴板恢复
1. 在任意应用中复制一段文字 "ORIGINAL"
2. 光标聚焦到文本编辑器的输入框
3. 按住热键说一句话，松开
4. 等待文本注入完成
5. 新开一个文本框，按 Cmd+V
6. **预期**: 粘贴出来的是 "ORIGINAL"（旧剪贴板内容被恢复）
7. **验证**: 输入框有识别文本 + 剪贴板恢复了原内容

### 用例 2.2: 注入失败时文本留在剪贴板
1. 复制 "ORIGINAL" 到剪贴板
2. 光标聚焦到一个**不可编辑**的地方（如桌面、Finder 窗口）
3. 按住热键说一句话，松开
4. 查看通知/状态栏消息
5. 按 Cmd+V 到一个文本编辑器
6. **预期**: 状态显示"已粘贴到剪贴板"，粘贴出来的是识别文本（不是 "ORIGINAL"）
7. **验证**: 注入失败时文本保留在剪贴板供手动粘贴

## 3. #107 ESC 打断

### 用例 3.1: ESC 打断后文本在剪贴板
1. 复制 "ORIGINAL" 到剪贴板
2. 按住热键说一句话
3. 在识别过程中按 ESC
4. 按 Cmd+V 到文本编辑器
5. **预期**: 粘贴出来的是刚才说的识别文本（不是 "ORIGINAL"）
6. **验证**: ESC 打断后文本写入剪贴板，旧内容不恢复

### 用例 3.2: ESC 打断后不自动注入
1. 光标聚焦到文本编辑器
2. 按住热键说一句话
3. 按 ESC 打断
4. **预期**: 文本编辑器中**没有**新增文字
5. **验证**: ESC 只写剪贴板不注入

### 用例 3.3: ESC 打断的历史记录
1. 按住热键说一句话，按 ESC 打断
2. 打开 Type4Me 设置 > 历史记录
3. **预期**: 能看到这条记录，状态标记为已取消/aborted
4. **验证**: 历史完整，方便回溯

## 4. #113a 麦克风选择

### 测试前提
- 至少连接 2 个音频输入设备（如笔记本内置麦克风 + 耳机/外接麦克风）

### 用例 4.1: 设置界面显示可用设备
1. 打开 Type4Me 设置
2. 找到麦克风选择下拉
3. **预期**: 列表中显示"系统默认"和所有可用麦克风设备名
4. **验证**: 设备列表完整，能区分内置和外接

### 用例 4.2: 切换麦克风
1. 插上耳机（带麦克风）
2. 在设置中选择笔记本内置麦克风
3. 按住热键说话，对着笔记本说而不是耳机
4. **预期**: 能正常识别（从内置麦克风采集）
5. **验证**: 切换生效

### 用例 4.3: 选中设备被拔掉后 fallback
1. 选择外接麦克风
2. 拔掉外接麦克风
3. 按住热键说话
4. **预期**: 自动 fallback 到系统默认设备，正常工作
5. **验证**: 不崩溃，不卡死

### 用例 4.4: 重启后记住选择
1. 选择一个非默认麦克风
2. 退出 Type4Me，重新启动
3. 打开设置检查
4. **预期**: 仍然选中之前的麦克风
5. **验证**: UserDefaults 持久化正确

## 5. #113b 鼠标按键热键

### 测试前提
- 有带额外按键的鼠标（如罗技，有中键和侧键）

### 用例 5.1: 热键录制器捕获鼠标键
1. 打开 Type4Me 设置 > 热键
2. 点击录制热键按钮
3. 按鼠标中键（或侧键）
4. **预期**: 录制器捕获到鼠标按键，显示 "Mouse 3" / "Mouse 4" 等
5. **验证**: 鼠标按键被正确识别和显示

### 用例 5.2: 鼠标键 Hold 模式录音
1. 将热键设为鼠标侧键（Hold 模式）
2. 按住侧键说话
3. 松开侧键
4. **预期**: 正常录音、识别、注入
5. **验证**: 完整的录音-识别-注入流程正常

### 用例 5.3: 鼠标键 Toggle 模式录音
1. 将热键设为鼠标侧键（Toggle 模式）
2. 按一下侧键开始录音
3. 说话
4. 再按一下侧键停止
5. **预期**: 正常识别和注入
6. **验证**: Toggle 模式与鼠标键配合正常

### 用例 5.4: 键盘热键不受影响
1. 设置一个键盘热键（如 Option 键）
2. 用键盘热键录音
3. **预期**: 和之前一样正常工作
4. **验证**: 向后兼容

### 用例 5.5: 多按键鼠标区分不同侧键
1. 录制一个热键为鼠标按键 4（前侧键）
2. 录制另一个热键为鼠标按键 5（后侧键）
3. 分别按两个键
4. **预期**: 两个热键各自独立工作
5. **验证**: 不同鼠标按键能正确区分
