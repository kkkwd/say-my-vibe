## 复核要点

1. **冲突风险词** (建议谨慎添加):
   - `cloud → Claude`: 在讨论云计算时会误伤
   - `view → Vue`: 日常英语太常见
   - `rust → Rust`: 讨论生锈时冲突 (极少见)
   - `bun → Bun`: 讨论面包时冲突 (极少见)
   - `dino → Deno`: 讨论恐龙时冲突
   - `dolly → DALL-E`: 人名冲突
   建议: 这些词只加热词 boost，不加映射

2. **缩写读法映射**: 很多 `X.Y.Z. → XYZ` 的映射取决于 ASR 引擎输出格式，导入前建议先测试 ASR 对字母缩写的实际输出

3. **大小写映射**: 如 `javascript → JavaScript`，取决于 ASR 是否本身能正确大小写，可能不需要

4. **向量数据库名** (Pinecone, Weaviate, Chroma, Milvus): AI 圈高频但 ASR 基本不认识，映射优先级高

## 各引擎热词配置策略

### 火山引擎 (bigmodel_async)

**机制**: 请求时传 `context.hotwords`，每个词带 `scale` 权重 (当前硬编码 5.0)。
**限制**: 词必须在模型词表里，不在的话 boost 无效。
**适合**: 常见英文词的正确拼写偏好 (如 Claude vs cloud)，中文同音词。
**不适合**: 新造词 (vibe coding)、冷门专有名词 (Weaviate)。

建议热词 (高价值，模型大概率认识):
```
Claude, GPT, ChatGPT, OpenAI, DeepSeek, Gemini, Anthropic,
TypeScript, JavaScript, Python, React, Docker, Kubernetes,
API, JSON, WebSocket, GraphQL, GitHub, Copilot
```

### SenseVoice (本地 Python)

**机制**: 当前代码未传热词，所有纠正靠 Snippets 后处理。
**适合**: 无 (需要代码改动才能支持)。

### SherpaOnnx (本地)

**机制**: `hotwords_buf` 传入，格式为每行 `word score`，如 `Claude 5.0`。支持运行时通过 `reset(hotwords:)` 动态更新。还有 `HomophoneReplacer` 接口。
**限制**: 取决于模型 (Zipformer/Paraformer) 的词表覆盖。
**适合**: 和火山引擎类似，常见词的优先选择。

### Snippets (后处理映射) — 主力方案

**机制**: 正则替换，支持 CJK/ASCII 混排间的空格容差。对所有引擎生效。
**无限制**: 不受模型词表约束。
**适合**: 所有英文误识别纠正，特别是新造词、专有名词、多对一映射。

**配置优先级**:
1. 先上 Snippets 映射 (确定有效)
2. 再酌情加热词 boost (可能有效，不会有害)
3. 缩写类映射 (如 `L.L.M. → LLM`) 需要先测试 ASR 实际输出格式再决定

## 测试方法

1. **逐句测试**: 每句朗读两遍 (正常语速 + 略慢)，记录 ASR 输出
2. **对比引擎**: 同一句分别用火山引擎和 SenseVoice 测试
3. **标记结果**: 每个目标词标记 ✓ (正确) / ✗ (错误，记录实际输出)
4. **分析模式**: 汇总错误模式，确认哪些需要热词 boost，哪些需要映射
5. **添加映射后复测**: 导入映射后，对之前失败的句子复测验证
